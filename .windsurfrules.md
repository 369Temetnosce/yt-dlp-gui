## ğŸ”§ Code Standards & Conventions

### Python Style

- **Type Hints:** Required on all function signatures
  ```python
  def validate_url(url: str) -> bool:
      ...
  
  def fetch_metadata(url: str) -> dict[str, Any]:
      ...
  ```

- **Naming Conventions:**
  - Classes: `PascalCase` (e.g., `DownloadManager`)
  - Functions/methods: `snake_case` (e.g., `start_download()`)
  - Constants: `UPPER_SNAKE_CASE` (e.g., `DEFAULT_OUTPUT_PATH`)
  - Private methods: `_snake_case` (e.g., `_parse_progress()`)

- **Docstrings:** Use Google-style docstrings for public methods
  ```python
  def fetch_metadata(self, url: str) -> dict:
      """Fetch video metadata from YouTube.
      
      Args:
          url: YouTube video URL
      
      Returns:
          dict with keys: title, duration, formats, thumbnail_url
      """
  ```

### Threading

- **UI Freezing Prevention:** Always use `QThread` for long operations
  ```python
  # âœ… Correct
  thread = QThread()
  worker = DownloadWorker(url, format, quality)
  worker.moveToThread(thread)
  thread.started.connect(worker.download)
  worker.finished.connect(thread.quit)
  thread.start()
  
  # âŒ Wrong
  download_manager.start_download(...)  # Blocks UI
  ```

- **Signal Emissions:** Emit signals from worker thread, update UI from main thread
  ```python
  # In worker thread:
  self.progress_updated.emit(percentage, speed, eta)
  
  # In main window:
  self.download_manager.progress_updated.connect(self.on_progress_update)
  ```

### Error Handling

- **Never raise exceptions to user.** Catch all exceptions, return error in dict/signal:
  ```python
  # âœ… Correct
  def fetch_metadata(self, url: str) -> dict:
      try:
          ...
      except Exception as e:
          logging.error(f"Metadata fetch failed: {e}")
          return {"error": "Could not fetch video info. Check your connection."}
  
  # âŒ Wrong
  def fetch_metadata(self, url: str) -> dict:
      ...  # Let exception propagate
  ```

- **User-Friendly Messages:** No jargon, always suggest action
  ```python
  # âœ… Good
  "This URL doesn't look like a YouTube link. Please check and try again."
  
  # âŒ Bad
  "Invalid regex match on extractor pattern"
  ```

### Logging

- **Always use `logging` module, never `print()`**
  ```python
  import logging
  logging.info("Download started: My Video.mp4")
  logging.error("Network timeout during metadata fetch")
  ```

- **Log Level Usage:**
  - `DEBUG`: Detailed info (yt-dlp command executed, etc.)
  - `INFO`: Important events (download started/completed)
  - `WARNING`: Issues but recoverable (ffmpeg missing)
  - `ERROR`: Failures affecting user (download failed, can't write file)

### File Organization

- **Path Handling:** Use `pathlib.Path` for cross-platform compatibility
  ```python
  from pathlib import Path
  
  # âœ… Correct (works on Windows, macOS, Linux)
  output_dir = Path.home() / "Downloads" / "yt-dlp"
  
  # âŒ Wrong (hardcoded slashes)
  output_dir = "~/Downloads/yt-dlp"
  ```

## ğŸ”’ Security & Safety

### Input Validation

- **URL Validation:** Regex check for `youtube.com` or `youtu.be`
  ```python
  import re
  pattern = r'(youtube|youtu\.be)'
  return bool(re.search(pattern, url))
  ```

- **File Path Sanitization:** Remove illegal characters
  ```python
  # Remove: < > : " / \ | ? *
  illegal_chars = '<>:"/\\|?*'
  for char in illegal_chars:
      filename = filename.replace(char, '_')
  ```

### Process Safety

- **Subprocess Security:** No shell injection
  ```python
  # âœ… Correct (list, no shell=True)
  subprocess.run(['yt-dlp', '--dump-json', url], ...)
  
  # âŒ Wrong (shell=True allows injection)
  subprocess.run(f'yt-dlp --dump-json {url}', shell=True, ...)
  ```

- **Timeout Protection:** Prevent hangs
  ```python
  subprocess.run(..., timeout=10)  # 10-second max wait
  ```

### Data Privacy

- **No Telemetry:** Don't collect user data
- **No Cookies Storage (MVP):** Don't save browser cookies
- **Local-Only:** App works entirely offline after yt-dlp installed
- **Config Storage:** Only user preferences (save path, last URL, history)

## ğŸ“ Common Error Scenarios & Handling

### Error Case 1: Invalid URL

**User Action:** Paste non-YouTube URL  
**Expected:** Download button disabled, error message shown  
**Message:** "This URL doesn't look like a YouTube link. Please check and try again."  
**Action:** Focus on URL field, suggest valid format

```python
if not YTDLPWrapper.validate_url(url):
    self.show_error_dialog("Invalid URL", "This URL doesn't look like a YouTube link...")
    self.download_button.setEnabled(False)
```

### Error Case 2: yt-dlp Not Installed

**Trigger:** App startup  
**Expected:** Show warning dialog with install link  
**Message:** "yt-dlp is not installed. Download from: https://github.com/yt-dlp/yt-dlp"  
**Allow:** User to dismiss warning, but Download button stays disabled

```python
def __init__(self):
    if not YTDLPWrapper.check_yt_dlp_installed():
        self.show_warning_dialog("yt-dlp Not Found", "Install from: [link]")
        self.download_button.setEnabled(False)
```

### Error Case 3: Network Timeout

**Trigger:** Metadata fetch takes > 10 seconds  
**Expected:** Show timeout error, offer retry  
**Message:** "Could not fetch video info. Check your internet connection and try again."  
**Action:** Clear input, focus on URL field for retry

```python
try:
    metadata = ytdlp_wrapper.fetch_metadata(url)
except TimeoutError:
    self.show_error_dialog("Timeout", "Could not fetch video info...")
    self.url_input.clear()
    self.url_input.setFocus()
```

### Error Case 4: Video Unavailable

**Trigger:** Metadata fetch returns empty/error  
**Expected:** Clear message  
**Message:** "This video is not available. Check if it's public or try another."  
**Action:** Keep URL visible for user to try another link

```python
metadata = ytdlp_wrapper.fetch_metadata(url)
if not metadata or 'error' in metadata:
    self.show_error_dialog("Video Not Available", "Check if it's public...")
```

### Error Case 5: File System Error (Permission Denied)

**Trigger:** Can't write to output folder  
**Expected:** Clear message with suggestion  
**Message:** "Cannot save to this location. Check folder permissions or choose another."  
**Action:** Suggest Browse to change folder

```python
try:
    Path(output_path).mkdir(parents=True, exist_ok=True)
except PermissionError:
    self.show_error_dialog("Permission Denied", "Cannot save to this location...")
    self.browse_button.click()  # Open folder picker
```

### Error Case 6: Disk Space Low

**Trigger:** Free space < 100MB  
**Expected:** Show warning before download  
**Message:** "Low disk space (XX MB available). Download may fail."  
**Action:** Warn, but allow user to proceed or change location

```python
import shutil
free_space = shutil.disk_usage(output_path).free / (1024**2)
if free_space < 100:
    reply = self.show_warning_dialog("Low Disk Space", f"{free_space:.1f} MB available. Continue?")
```
